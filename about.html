How It Works

A complete walkthrough of the architecture, dataset, and explainability layer.

01

The Dataset

We train on the NIH Chest X-Ray14 dataset — one of the largest publicly available chest X-ray collections. It contains 112,120 frontal-view X-rays from 30,805 unique patients, labelled with up to 14 thoracic pathology findings using NLP-mined annotations from radiology reports.

112,120 images30,805 patients 14 pathology labelsMulti-label classification
02

DenseNet-121 Backbone

The model is a DenseNet-121 pre-trained on ImageNet, fine-tuned for multi-label chest pathology classification — the same architecture used in the seminal CheXNet paper (Rajpurkar et al., 2017).

Each dense block connects every layer to every other layer in a feed-forward fashion, enabling strong gradient flow and feature reuse — critical for detecting subtle radiology findings.

03

Multi-Label Training

A patient can have multiple simultaneous conditions. We model this with Binary Cross-Entropy with Logits Loss and class-frequency-aware positive weights to handle severe label imbalance (e.g., Hernia appears in < 0.2% of images).

Performance is measured via mean AUC-ROC across all 14 classes, consistent with clinical literature benchmarks.

04

Explainable AI — Grad-CAM

Gradient-weighted Class Activation Mapping (Grad-CAM) makes the model interpretable. For each prediction, it:

  1. Computes the gradient of the predicted class score with respect to the final convolutional feature maps.
  2. Pools these gradients globally to get channel importance weights.
  3. Produces a weighted combination of activation maps and applies ReLU, resulting in a coarse localisation heatmap.
  4. Overlays this heatmap on the original X-ray using a jet colormap — red = high activation (where the model "looked"), blue = low.

This allows clinicians to verify the model's spatial reasoning and build trust or flag errors — going far beyond a plain probability score.
